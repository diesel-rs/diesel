Flags for the json_valid function

These flags define what is meant by โwell-formedโ JSON when validating.

The following bits are currently defined:

  • 0x01 โ The input is text that strictly complies with canonical RFC-8259 JSON, without any extensions.
  • 0x02 โ The input is text that is JSON with JSON5 extensions.
  • 0x04 โ The input is a BLOB that superficially appears to be JSONB.
  • 0x08 โ The input is a BLOB that strictly conforms to the internal JSONB format.

By combining bits, the following useful values can be derived:

  • Rfc8259Json (1) โ X is RFC-8259 JSON text
  • Json5 (2) โ X is JSON5 text
  • JsonbLike (4) โ X is probably JSONB
  • Rfc8259JsonOrJsonb (5) โ X is RFC-8259 JSON text or JSONB
  • Json5OrJsonb (6) โ X is JSON5 text or JSONB (recommended for most use cases)
  • JsonbStrict (8) โ X is strictly conforming JSONB
  • Rfc8259JsonOrJsonbStrict (9) โ X is RFC-8259 or strictly conforming JSONB
  • Json5OrJsonbStrict (10) โ X is JSON5 or strictly conforming JSONB
