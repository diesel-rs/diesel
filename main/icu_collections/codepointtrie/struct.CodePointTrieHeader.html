This struct contains the fixed-length header fields of a CodePointTrie.

ยงSafety Invariant

The trie_type field must not change after construction.

(In practice, all the fields here remain unchanged during the lifetime of the trie.)

Fieldsยง

ยงhigh_start: u32

The code point of the start of the last range of the trie. A range is defined as a partition of the code point space such that the value in this trie associated with all code points of the same range is the same.

For the property value data for many Unicode properties, often times, high_start is U+10000 or lower. In such cases, not reserving space in the index array for duplicate values is a large savings. The โhighValueโ associated with the high_start range is stored at the second-to-last position of the data array. (See impl_const::HIGH_VALUE_NEG_DATA_OFFSET.)

ยงshifted12_high_start: u16

A version of the high_start value that is right-shifted 12 spaces, but is rounded up to a multiple 0x1000 for easy testing from UTF-8 lead bytes.

ยงindex3_null_offset: u16

Offset for the null block in the โindex-3โ table of the index array. Set to an impossibly high value (e.g., 0xffff) if there is no dedicated index-3 null block.

ยงdata_null_offset: u32

Internal data null block offset, not shifted. Set to an impossibly high value (e.g., 0xfffff) if there is no dedicated data null block.

ยงnull_value: u32

The value stored in the trie that represents a null value being associated to a code point.

ยงtrie_type: TrieType

The enum value representing the type of trie, where trie type is as it is defined in ICU (ex: Fast, Small).
