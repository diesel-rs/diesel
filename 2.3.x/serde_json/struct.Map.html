pub fn sort_keys(&mut self)

Sorts this mapโs entries in-place using strโs usual ordering.

If serde_jsonโs โpreserve_orderโ feature is not enabled, this method does no work because all JSON maps are always kept in a sorted state.

If serde_jsonโs โpreserve_orderโ feature is enabled, this method destroys the original source order or insertion order of this map in favor of an alphanumerical order that matches how a BTreeMap with the same contents would be ordered. This takes O(n log n + c) time where n is the length of the map and c is the capacity.

Other maps nested within the values of this map are not sorted. If you need the entire data structure to be sorted at all levels, you must also call map.values_mut().for_each(Value::sort_all_objects).
